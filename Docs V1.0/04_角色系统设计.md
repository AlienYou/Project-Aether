# 04_角色系统设计（Character System Design）

版本：v1.0

项目：Project Aether

引擎版本：Unity 2022.3.51f1c1

文档状态：正式版

---

# 1. 文档目标

定义角色系统（Character System）。

角色系统是整个Gameplay层的基础。

所有战斗单位：

* 玩家
* 怪物
* Boss
* NPC
* 召唤物

统一使用Character架构。

---

# 2. 设计原则

遵循：

组合优于继承

数据驱动

模块化

网络友好

可扩展

---

# 3. 系统定位

系统关系：

```text

↓

↓

↓

↓

```

角色系统是所有战斗系统的根节点。

# 4. 核心架构

Character

├── CharacterEntity

├── CharacterRuntime

├── CharacterConfig

├── CharacterController

├── CharacterStateMachine

├── CharacterAnimation

└── CharacterView

---

# 5. CharacterEntity

领域对象。

负责：

* 唯一ID
* 配置引用
* 阵营
* 类型

不负责：

* 移动
* 战斗
* 表现

---

```

# 7. Faction系统

```

```csharp

GuildB
```

# 8. CharacterRuntime

运行时对象。

负责：

* 当前状态
* 属性快照
* Buff列表
* 技能列表

---

配置数据。

来源：

```text

↓

↓

```

---

```text

```

# 10. CharacterView

表现层。

负责：

* 模型
* 特效
* 动画
* UI挂点

禁止：

业务逻辑

---

# 11. CharacterController

控制层。

负责：

* 输入处理
* 移动请求
* 技能请求

---

玩家：

PlayerController

---

AI：

AIController

# 33. 结论

Character是Project Aether所有战斗单位的统一抽象。

Player、Monster、Boss均基于Character构建。

通过组件化与状态机设计，实现工业级可扩展角色架构。
