# 02_框架设计（Framework Design）

版本：v1.0

项目：Project Aether

引擎版本：Unity 2022.3.51f1c1

文档状态：正式版

---

# 1. 文档目标

定义整个项目的基础框架。

负责：

* 生命周期管理
* 服务管理
* 事件系统
* 配置系统
* 资源系统
* 网络系统
* 场景系统

为所有业务模块提供基础能力。

---

# 2. 设计原则

遵循：

高内聚

低耦合

模块隔离

依赖倒置

数据驱动

可测试

---

# 3. Framework定位

整体结构：

```text
Game

├── Framework
├── Gameplay
├── UI
└── Editor
```

职责：

Framework

↓

提供基础能力

Gameplay

↓

实现业务逻辑

---

禁止：

Gameplay依赖具体实现。

允许：

Gameplay依赖接口。

---

# 4. Framework模块划分

```text
Framework

├── Core
├── Event
├── Config
├── Resource
├── Network
├── Scene
├── Save
├── Pool
├── Audio
└── Localization
```

---

# 5. Core模块

项目核心模块。

负责：

* 生命周期
* Service注册
* 初始化流程
* 全局上下文

核心接口：

```csharp
IGameService
IInitializable
IDisposable
```

---

# 6. 生命周期管理

统一生命周期：

```text
Register

↓

Initialize

↓

Start

↓

Update

↓

Dispose
```

---

禁止：

```csharp
Awake里处理业务
```

---

业务初始化：

统一进入：

```csharp
Initialize()
```

---

# 7. 服务架构

所有全局能力：

统一Service化。

例如：

```text
ConfigService

ResourceService

NetworkService

SceneService

AudioService
```

---

接口优先：

```csharp
IConfigService
```

而不是：

```csharp
ConfigManager
```

---

# 8. VContainer规范

采用：

VContainer

作为唯一DI框架。

---

注册：

```csharp
builder.Register<ConfigService>
(
Lifetime.Singleton
);
```

---

解析：

```csharp
class SkillSystem
{
    [Inject]
    private IConfigService _config;
}
```

---

禁止：

```csharp
ConfigService.Instance
```

---

# 9. EventBus设计

目标：

模块解耦。

---

例如：

```text
Boss死亡

↓

BossDeadEvent

↓

SettlementSystem
```

---

而不是：

```text
Boss

↓

直接调用Settlement
```

---

# 10. Event结构

事件必须：

不可变

只读

轻量

---

示例：

```csharp
public readonly struct BossDeadEvent
{
    public int BossId;
}
```

---

# 25. 风险分析

风险1：

Singleton泛滥

解决：

VContainer

---

风险2：

模块耦合

解决：

接口依赖

---

风险3：

FishNet侵入业务

解决：

Network Adapter
